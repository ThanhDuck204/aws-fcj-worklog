## Weekly Notes

This was the final week to complete the project and submit the workshop. The first two days (Monday, Tuesday) focused on finishing the remaining features and beginning the deployment process to AWS.

However, the deployment process encountered many unexpected errors — from service configuration issues and connection problems between components to production environment differences from the development environment. Wednesday was spent almost entirely debugging and fixing these issues.

By Thursday, the project was nearly complete with core features working stably. Since time was running out, I decided to submit the workshop in its current state rather than trying to fix more issues — accepting some minor remaining limitations and focusing on completing the report for an on-time submission.

---

## Challenges Encountered

* The deployment process to AWS encountered many unexpected errors not seen in the development environment.
* Configuration errors between AWS services during actual deployment differed from running locally.
* Tight schedule — insufficient time to thoroughly fix all issues that arose.
* Had to make a difficult decision between continuing to fix bugs or submitting the workshop on time.
* Deadline pressure as this was the final week to complete everything.

---

## Lessons Learned

* Deploying a project to a real environment always brings up issues not visible when running locally — better preparation is needed for this phase.
* Production debugging skills are crucial: reading logs, analyzing errors, and fixing issues quickly under time pressure.
* Knowing how to prioritize tasks: not every bug needs immediate fixing — assess the impact and decide what's most important to complete.
* Time management and self-discipline are key factors in completing a project on schedule.
* Accepting that a real-world project is never 100% perfect — knowing when to stop is just as important as trying to fix every bug.
