## Weekly Notes

This week marked an important milestone as I started integrating real AWS services into the project. After weeks of architecture design, theoretical study, and preparation, it was time to put that knowledge into practice.

Integrating AWS Cognito for authentication, S3 for storage, and DynamoDB for the database were the first steps. However, the process wasn't entirely smooth — many errors arose during connection setup, requiring multiple rounds of debugging and configuration adjustments. By the end of the week, I had partially completed the AWS integration, with basic connections running stably.

Mid-week, I took a day off for health reasons, which slightly slowed down progress, but things were still on track.

At the end of the week, I went to the company to attend an event — a great opportunity to socialize, learn, and recharge after intense work days.

---

## Challenges Encountered

* When starting the AWS services integration, configuring connections and handling errors was quite complex — requiring patient step-by-step debugging.
* Adjusting the codebase to fit the new architecture with AWS services required reviewing many parts — from configuration and data handling to the overall operational flow.
* A mid-week sick day caused some disruption to progress, requiring catch-up work in the following days.
* Integrating real services was very different from running with mocks — there were many edge cases and unexpected environment errors.

---

## Lessons Learned

* Integrating real AWS services into the project showed a clear difference between running with mock services and running with real services.
* Patience is key when handling connection errors — most issues came from incorrect configuration or missing IAM permissions.
* Cognito makes authentication quick to set up, but understanding the flow is essential for proper integration.
* DynamoDB with its NoSQL design is suitable for an MVP, but table and query patterns need careful design from the start.
* Combining multiple AWS services requires a holistic view of the architecture to avoid configuration conflicts.
* Company events are valuable opportunities to learn practical experience and expand professional networks.
